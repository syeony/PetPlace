# 250724 ERD 특강

## 1. DB 설계
- **목표**: 웹사이트 설계와 클라이언트 대응 방법
- **핵심**: 이용객에 맞는 추가 기능

### 기본 원칙
- **무결성 보장**: 어떻게 응용 프로그램에서 무결성을 지킬 것인가
- **ER(Entity Relationship) 다이어그램**: 데이터 무결성을 위한 설계 도구
- **정규화**: 중복을 없애기 위한 데이터 구조 최적화

## 2. 데이터 무결성 확보 방법

### 테이블 설계 시 고려사항
- **무결성 제약**: 잘못된 값이 자동으로 차단되도록
- **참조무결성**: 서로 다른 테이블 간의 관계에서 데이터 일관성 유지
- **개체무결성**: 개별 엔티티 단위의 정확성 보장
- **영역무결성**: 도메인 내에서 올바른 데이터 형식 유지

## 3. 키(Key) 개념

### 기본 키(Primary Key)
- 각 레코드를 고유하게 식별하는 키
- NULL 값 불가
- 중복 불가

### 외래 키(Foreign Key)
- 다른 테이블의 기본 키를 참조하는 키
- 테이블 간 관계를 나타냄
- 참조 무결성 제약에 따라 관리

## 4. 실무 적용 사항

### VARCHAR vs CHAR
- **VARCHAR**: 가변 길이 문자열 (메모리 효율적)
- **CHAR**: 고정 길이 문자열 (처리 속도 빠름)
- **TEXT**: 긴 텍스트 데이터용

### 인덱싱 전략
- **Key를 통한 검색 최적화**
- **복합 인덱스**: 여러 컬럼을 조합한 인덱스
- **성능 향상**을 위한 핵심 요소

## 5. DB 엔터티 관계도

### UUID vs 일반 키
- **UUID**: 전역적으로 고유한 식별자
- **장점**: 분산 시스템에서 중복 없음
- **단점**: 크기가 크고 성능 영향
- **Auth 시스템과의 연계**: 사용자 인증과 권한 관리
