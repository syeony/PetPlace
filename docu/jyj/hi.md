# D104팀 3주차 회고 정리
📊 번다운 차트를 보면서
- 정유진: 이번주는 지난주와 달리 여러 계획을 세워서 시작했습니다. 순차적으로 진행하는 것은 좋았으나, 생각보다 기획에서 시간을 많이 사용하여 모양이 아쉽습니다. 다음주 부터는 적당히 정하는 것도 좋을 것 같습니다!!!!!
- 오승연: 번다운차트가 마지막에 끝까지 내려가지 못한 이유는 월요일에 일주일치 일을 정할때 너무 무리하게 정해서인 것 같습니다. 다음주에는 조금 더 꼼꼼하게 시간분배를 하여 적당한 양을 할당해야할 거 같습니다.
- 조경호: 번다운 차트를 작성할 때 많은 양의 업무를 작성하기도 했고, 에픽을 사용하지 않아 다음 주에는 에픽을 활용하여 진행해야 될 것 같습니다.
- 이도형: 처음 스프린트를 시작하는 날 업무를 구체적으로 떠올리지 못해서 중간중간 업무량이 늘어난 거 같은데 처음 업무를 정할 때 조금 더 많이 생각해서 정했으면 좋을 것 같고 이번 주에는 업무를 모두 끝마치지 못했는데 다음 주에는 모든 업무를 마무리할 수 있을 정도의 양으로 배정해야 할 것 같고 다음주에는 에픽도 정해서 하면 좋을것같다
- 김민: 첫째날과 둘째날에서는 우리가 어떤 일을 해야하는지 몰라서 추가와 진행이 반복되었고, 이후 할 수 있는 양을 초과하여 할당하는 등의 이슈가 있었다. 이는 마무리되지 않은 계단식 차트를 보여주게 되었고, 차주에는 해결할 수 있는 만큼의 작업을 할당해야겠다
- 송정현: 스프린트 첫날에 할 일을 모두 넣으려고 했지만, 중간에 또 생각지 못한 일이 추가되어 그래프가 초반에 올라간 것 같습니다. 모든 일정을 다 완료하지 못한 것은 할 일을 너무 크게 잡은 탓인 것 같습니다. 일의 볼륨을 더 꼼꼼하게 살펴서 계획하면 좋을 것 같습니다.

📋 기획안 발표 피드백을 보면서
- 정유진: 저희의 기획이 어떻게 하면 더 설득이 될지 고민할 수 있었던 시간이었던 것 같습니다. 우려하던 부분에 대해서 피드백을 주시기도 하셨고, 좋게 봐주신 부분도 많아서 좋았습니다. 앞으로 개발할때 어떤 부분을 챙기면 좋을지 더 고민해 볼 수있었던 시간입니다.
- 오승연: 생각보다 많은 분들이 장문으로 다양한 의견을 남겨주어서 감사했습니다. 예상했던 피드백도 있었고 예상치 못했던 피드백도 있어서 꼼꼼히 읽어보고 적용할 것은 적용해보도록 해야겠습니다.
- 이도형: 발표에서 강조한 내용들이 발표를 들은 사람들에게 어느 정도 전달이 된 거 같아서 기분이 좋지만 확실히 우리 조가 우려했던 부분들을 다른 사람들도 생각을 하는 것 같아서 보완해야 할 것 같다고 생각한다
- 김민: 팀원끼리 회의를 거치며 예상했던 문제점들도 피드백으로 많이 받았지만, 예상하지 못했던 부분과 아이디어 구체화 중 덜어냈던 파트들이 개선 피드백으로 오는 것을 보며 다양한 생각이 들었다
- 송정현: 정말 다양한 의견들을 남겨주셔서 감사했습니다. 칭찬한 점도 있고 비판적인 내용도 있었지만 저희가 기획한 내용에서 크게 벗어나는 의견에 휘둘릴 필요는 없을 것 같습니다.
- 조경호: 피드백 선정 - "이런 서비스가 있다면 반려 동물들을 키우고 있는 사람들이 안심하고 여행을 갈 수 있을 것 같아요!! 반려 동물들을 맡기기에 신뢰도가 부족해 서비스를 만든다고 하였는데 당근 마켓에서도 이런 사람들을 구인하는 글을 볼 수 있습니다. 하지만 이렇게 혼자가 되도록 유도하는 범죄들도 많은데 이건 어쩔 수 없겠죠? ㅠㅠㅠ 더해서 병원 등의 후기를 믿을 수 없다고 했는데 요새 네이버 후기는 영수증 인증을 해야하더라고요? 같은 후기 시스템이라면 비슷할 수 있지 않을까 싶어요... 그리고 누군가의 실수로 동물이 큰일나게 된다면 책임은 어떻게 처리되는지 생물이라서 고민이 되지 않을까 싶어요!! 화이팅이예요!!" → 신뢰도 문제는 중요한 문제이기에 인증 시스템 그리고 신고 시스템을 도입하여 이를 활용한 제제를 통해 관리 시스템을 구축해야 할 것 같습니다.

🔄 KPT 회고
- 정유진

Keep: 열정적으로 작업에 참여하는 태도가 너무 좋습니다. 아자아자 힘낼 수 있었습니다!
Problem: 처음에 많은 과업을 작성해 번다운 차트가 아쉽습니다. 다음 주엔 스토리 포인트 부터 잘 결정해보면 좋을 것 같습니다.
Try: 적절한 양의 스토리를 각자 할당해 성실하게 할 일을 하면 너무 좋을 것 같습니다!!

- 오승연

Keep: 다들 열심히 하는건 유지하되(어제도 다들 새벽2시에 잤음)
Problem: 번다운차트 관리의 필요성을 느꼈다.
Try: 다음주부턴 본격적으로 개발에 들어가면서 개인의 할당된 양을 완료한다면 그때그때 완료로 옮겨 예쁜 모양의 번다운차트를 만들것이다.

- 이도형

Keep: 그래도 갈수록 차트가 내려가는건 유지하면 좋을것같다
Problem: 중간중간 업무량이 늘지않도록 이제는 미리미리 정하면 좋겠다
Try: 번다운차트 모양 이쁘게해보고싶습니다

- 김민

Keep: 전주와 달리 해야할 일을 파악하는 능력이 늘었고, 작업을 처리하는 과정이 잡혀가고있다
Problem: 해야할 작업을 과하게 할당하거나 할당 받은 작업을 끝내지 못하는 것은 노력할 필요가 있다
Try: 이제부터 체계화된 작업이 시작이 될텐데, 각자 기능 브랜치와 스토리를 잘 관리해보자

- 송정현

Keep: 발표 기획에 대한 내용을 꼼꼼하게 고민해보고 준비한 점이 좋았습니다.
Problem: 할 일을 너무 많이 잡아둬서 계획한 일을 다 끝내지 못한 점이 아쉽습니다.
Try: 계획을 조금 더 면밀하게 분석해서 어느 날에 무엇을 할 지 생각하고 잡으면 좋을 것 같습니다.

- 조경호

Keep: 프로젝트 기획을 하면서 기능에 관한 토론 과정에서 서로의 의견을 경청하고, 자신의 주장을 설명하는 과정이 굉장히 좋았다고 생각합니다. 또한 디테일과 팀을 위해 활동 시간 외에도 열심히 업무를 해주셔서 좋았습니다.
Problem: 디테일을 위해 볼륨을 너무 키우다 보니 업무량, 회의 시간이 길어졌던 것 같습니다.
Try: 작업을 시작하게 될 것 같은데 깃 컨벤션에 맞춘 프로젝트 관리와 소통이 잘 이루어지면 좋겠습니다.
